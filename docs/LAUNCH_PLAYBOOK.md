# Agent Radar launch and discovery playbook

This page contains the repository metadata and distribution assets that cannot be configured through committed files alone. It is intentionally practical: complete the repository settings first, then share releases where AI builders already gather.

## Repository metadata

Use this description under **About → Edit repository details**:

> Daily GitHub radar for trending LLM, AI Agent, MCP and RAG repositories—ranked by 7-day Star growth and delivered to Feishu, Slack, Telegram, WeCom and Discord.

Add these focused GitHub Topics (maximum 20):

Topics are searchable and connect the project to GitHub's topic pages. Keep them accurate; irrelevant topic stuffing weakens trust.

## Social preview

GitHub recommends a 1280×640 image for the best rendering. This repository includes the source asset at [`assets/agent-radar-social-preview.svg`](../assets/agent-radar-social-preview.svg).

1. Export the SVG as a 1280×640 PNG under 1 MB.
2. Open **Settings → General → Social preview**.
3. Select **Edit → Upload an image** and upload the PNG.
4. Share the repository URL in a private message once to verify the card before a public launch.

## Sustainable distribution

- Publish meaningful GitHub Releases with a short demo and upgrade notes.
- Share major channel additions in the relevant provider community.
- Submit the project to accurate AI tooling directories and curated lists that accept automation tools.
- Turn ranking insights into useful posts, such as a monthly analysis of emerging Agent or MCP projects.
- Invite users to share screenshots and integrations through the **Showcase** issue template.
- Respond quickly to first-time contributors and label approachable work as `good first issue`.

Track outcomes in **Insights → Traffic** using referring sites, popular content, unique visitors, clones, and conversion to Stars or forks. Prefer steady qualified traffic over one-off impressions.

## Naming note

The current repository slug contains `awsome`. GitHub redirects old repository URLs after a rename, but renaming still affects clones, badges, documentation, and automation. Consider a future, separately planned rename to `agent-radar` or `awesome-llm-agent-radar`; do not combine it with unrelated changes.
